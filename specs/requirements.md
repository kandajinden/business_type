# REQ-1: 基本情報入力

### REQ-1.1: 事前入力フォーム
- ユーザーは診断開始前に以下3項目を入力する

  - **性別**：男性 / 女性 / その他（ラジオボタン）

# REQ-2: 診断フロー（30問）

### REQ-2.1: 問い種別
- **5択行動選択問**（24問）：5つの選択肢から1つを選ぶ
- **スライダー問**（6問）：0〜100のスライダーで位置を決める
- **6択問**（Q27のみ）：6つの選択肢をグリッド表示

### REQ-2.2: 問い構成
- Q1〜Q20：共通問（全ユーザー共通）
- Q21〜Q24：職種別問（選択した職種に応じて出し分け、6職種×4問=24問を保持）
- Q25：共通スライダー
- Q26〜Q29：ぶっちゃけ質問（共通）
- Q30：共通スライダー（最後の問い）

### REQ-2.3: スライダー配置
- Q5, Q10, Q15, Q20, Q25, Q30 がスライダー問
- 4問ごとに1問スライダーが入るリズム設計

### REQ-2.4: 直感2秒問
- Q3, Q8, Q13, Q18, Q24 が直感2秒問
- 画面上部に「⚡ 直感で。2秒で。」のラベルを表示

### REQ-2.5: 職種別ブリッジ
- Q20回答後、Q21の前に「ここからはあなたの職種に合わせた質問です。あなたの現場でのリアルな判断力を測定します。」のテキストを表示

### REQ-2.6: 中間フィードバック
- Q15（スライダー③）回答後に表示
- 15問時点の暫定戦闘力を計算して表示
- 「現在の戦闘力：推定○○,○○○ まだ本当の力は測りきれていません」

### REQ-2.7: プログレス表示
- 問題番号（Q1/30）は非表示
- 代わりに「戦闘力ゲージ」を表示（スカウター風）
- 問い種別ごとにゲージ増加量が異なる：シチュエーション問（+4%）> スライダー問（+3%）> 直感2秒問（+2%）
- ゲージが増えるたびにスキャンラインが走るアニメーション

### REQ-2.8: 回答データ記録
- 各問の選択肢（A〜F or スライダー値0〜100）
- 各問の回答時間（ミリ秒）
- SL3（Q15）の回答時間が10,000ms以上の場合、A（分析力）に+1ボーナス

### REQ-2.9: セッション管理・再開
- 診断開始時に `sessionId`（cuid）を発行し、各回答ごとにサーバーに途中保存する
- クライアント側でも `localStorage` に `sessionId` と回答済み問番号を保持する
- **リロード・通信断時**：最後に保存された設問の次から再開する
- **前問への戻り**：MVPでは戻れない（1方向のみ）
- **セッション失効**：最終回答から30分以上無操作の場合、セッションを失効する。再アクセス時は「前回の途中データがあります。続きから再開しますか？ / 最初からやり直しますか？」の選択肢を表示する
- **セッション失効後の再開を選択した場合**：失効フラグを解除し、最後の保存地点から再開する
- **やり直しを選択した場合**：既存セッションを論理削除し、新しいセッションを発行する

# REQ-3: スコアリング

### REQ-3.0: スコアリング定義の正本
- 本セクションおよび付録A〜Fの定義を実装・検証の唯一の基準とする
- 全30問の配点は [付録A](#付録a-全30問-設問文選択肢配点表) および [付録B](#付録b-職種別質問-設問文選択肢配点表) に収載
- 矛盾判定の詳細ルールは [付録C](#付録c-矛盾判定10ペア-詳細ルール) に収載

### REQ-3.1: 6軸スコア
- **L**（リーダーシップ）, **C**（コミュニケーション）, **G**（グリット）, **A**（分析力）, **I**（発想力）, **E**（実行力）
- 各選択肢のスコア配分は付録A・Bに全量定義
- スコア表記：`[L+2 G-1]` = Lに+2点、Gに-1点。絶対値は1〜3

**正規化の式：**
```
normalized = MAX(15, ((raw_score + 15) / 60) × 100)
```
- 理論上の素点範囲: -15 〜 +45
- 正規化後の範囲: 15 〜 100（最低値15にクランプ。レーダーチャートで「ゼロ」を出さない）

### REQ-3.2: 戦闘力スコア

**ベーススコア算出：**
```
raw_total = L_norm + C_norm + G_norm + A_norm + I_norm + E_norm
           （各軸15〜100、合計90〜600）
base_power = 5000 × (raw_total / 100) ^ 2.5
```

**最終計算式：**
```
final_power = clamp(5000, 1500000,
  base_power × confidence_bonus × all_rounder_bonus + legend_bonus
)
```

**演算順序（厳密）：**
1. `base_power` を算出
2. `confidence_bonus` を乗算（信頼度による倍率）
3. `all_rounder_bonus` を乗算（オールラウンダー倍率）
4. `legend_bonus` を加算（条件を満たす場合のみ +200,000）
5. 結果を `clamp(5000, 1500000)` でクリップ

**信頼度ボーナス倍率（confidence_bonus）：**

| 信頼度 | 倍率 |
|---|---|
| 10 | ×1.40 |
| 9 | ×1.30 |
| 8 | ×1.20 |
| 7 | ×1.10 |
| 6 | ×1.05 |
| 5以下 | ×1.00 |

**オールラウンダーボーナス倍率（all_rounder_bonus）：**

| 条件 | 倍率 |
|---|---|
| 全軸60以上 | ×1.35 |
| 全軸50以上（60未満含む） | ×1.25 |
| 全軸40以上（50未満含む） | ×1.15 |
| 上記いずれにも該当しない | ×1.00 |

> 判定は上から順に評価し、最初に該当した条件を適用する（最も高い倍率が優先）。

**伝説ボーナス（legend_bonus）：**

以下の**いずれか1つ以上**を満たす場合、+200,000 を加算：
- 条件A：信頼度10（矛盾0）かつ 全軸60以上
- 条件B：最高軸が90以上 かつ 信頼度8以上
- 条件C：Q27の自己診断と最終判定の歴史人物が一致 かつ 信頼度9以上

> `Q27一致` の判定：Q27で選んだ武将（A→信長, B→秀吉, C→家康, D→孔明, E→龍馬, F→武蔵）と、REQ-3.4で判定された歴史人物タイプが同一であること。

**想定分布：**

| 戦闘力レンジ | 出現率 | レンジ名 | レンジキー |
|---|---|---|---|
| 5,000〜9,999 | 5% | 覚醒前 | `awakening` |
| 10,000〜29,999 | 20% | 潜在戦力 | `potential` |
| 30,000〜99,999 | 40% | 実力者 | `capable` |
| 100,000〜299,999 | 22% | 精鋭 | `elite` |
| 300,000〜999,999 | 11% | トップランナー | `top_runner` |
| 1,000,000以上 | 2% | 伝説 | `legend` |

### REQ-3.5: 属性タイプ

SL1,SL2,SL4,SL5,SL6の5問で判定。**SL3はポテンシャル年収専用であり、属性判定に使用しない。**

**属性スコア計算式：**
```
突破スコア = SL1の値 + (100 - SL4の値)
戦略スコア = (100 - SL1の値) + (100 - SL5の値)
共鳴スコア = SL2の値 + SL4の値
革新スコア = SL5の値 + SL1の値
求道スコア = (100 - SL5の値) + SL6の値
```

最もスコアが高い属性を適用する。同率の場合、上記リストの上にあるものを優先する。

| 属性 | 属性キー | キャッチコピー |
|---|---|---|
| 突破 | `breakthrough` | 壁があるなら壊す。道がないなら作る。 |
| 戦略 | `strategy` | 勝てる場所で、勝てるタイミングで、勝てる方法で。 |
| 共鳴 | `resonance` | 1人の100歩より、100人の1歩。 |
| 革新 | `innovation` | 正解を探すな。正解を作れ。 |
| 求道 | `mastery` | 深さで勝負する。極めた者だけが見える世界がある。 |

### REQ-3.7: 同年代偏差値

**初期（データ蓄積前）：** 年齢別想定分布テーブルを使用。

| 年齢グループ | 年齢グループキー | 想定平均戦闘力 | 標準偏差 |
|---|---|---|---|
| 18〜22歳 | `18-22` | 40,000 | 20,000 |
| 23〜27歳 | `23-27` | 65,000 | 28,000 |
| 28〜34歳 | `28-34` | 95,000 | 35,000 |
| 35〜44歳 | `35-44` | 115,000 | 40,000 |
| 45〜54歳 | `45-54` | 105,000 | 42,000 |
| 55〜65歳 | `55-65` | 90,000 | 38,000 |

**偏差値算出：**
```
z_score = (final_power - mean) / std_dev
percentile_rank = (1 - Φ(z_score)) × 100   // 上位○%
```
> Φ は標準正規分布の累積分布関数。

**成熟期（1万件以上）：** 実データから mean/std_dev を年齢グループごとに再計算して切替。

- 診断完了時に `resultId`（cuid）と `shareToken`（ランダム文字列、英数字16文字）を発行する
- **通常の結果画面URL**：`/result/{sessionId}` — 本人セッションでのみ閲覧可能（セッションCookieまたはlocalStorageの`sessionId`と一致する場合のみ表示）
- **シェア用URL**：`/s/{shareToken}` — 誰でも閲覧可能
- **シェア用URLの公開範囲**：戦闘力スコア、人物タイプ、属性タイプ、レーダーチャート、信頼度、同年代偏差値
- **シェア用URLで非公開**：年齢、性別、職種、回答内容、ポテンシャル年収、向いてる仕事、ボーナス詳細
- LINE友だち追加時は `resultId` または `shareToken` を LIFFまたはクエリパラメータで引き継ぎ、同一診断結果に紐づけて詳細配信・再診断解放判定に利用する

# 付録D: 結果コピー文言

## D-1: 歴史人物 結果コピー

### 織田信長型（I最高）
- **キャッチ**: 「ルールは壊すためにある。お前が次のルールだ。」
- **解説**: 既存の常識に縛られない。「なぜそうしなきゃいけないの？」が頭の中に浮かぶタイプ。ゲームそのものを書き換える、それがあなたの戦い方。
- **注意点**: 変革のスピードに周囲がついてこれないことがある。3秒だけ相手の立場で考える習慣を持つと、革新はもっと遠くまで届く。

### 豊臣秀吉型（C最高）
- **キャッチ**: 「敵すら味方にする。お前の最大の武器は"人"だ。」
- **解説**: 人の心を動かす天才。初対面でも相手の懐に入れる。「この人と一緒に仕事したい」と思わせる空気を自然と作り出す。
- **注意点**: 人に合わせすぎて自分の本音が見えなくなることがある。「で、俺はどうしたいんだっけ？」と自分に問い直す時間を意識的に作ると、リーダーシップがもう一段上がる。

### 徳川家康型（G最高）
- **キャッチ**: 「諦めなかった者だけが見る景色がある。お前はその景色を知っている。」
- **解説**: 諦めるという選択肢を持っていない。派手さはないかもしれない。でも結果を出す人間は、最後まで立っていた人間だ。
- **注意点**: 粘り強さが裏目に出ると「損切りできない」になる。「耐えるフェーズか、撤退すべきフェーズか」をデータで判断する癖をつけると、粘りがさらに効率的になる。

### 諸葛孔明型（A最高）
- **キャッチ**: 「勝負は始まる前に決まっている。お前の頭脳がそれを証明する。」
- **解説**: 感情ではなく事実で判断する。数字と根拠を求める。その慎重さが致命的なミスを未然に防いでいる。
- **注意点**: 分析に時間をかけすぎてチャンスを逃すことがある。「80%の確度で動く勇気」を持つと、頭脳に"スピード"という翼がつく。

### 坂本龍馬型（L最高）
- **キャッチ**: 「世界を変えるのは、いつだって"最初の1人"だ。」
- **解説**: 誰もやらないことを最初にやる。ビジョンを語り、人を巻き込む。リーダーシップはポジションから来るものじゃない。ビジョンから来るものだ。
- **注意点**: ビジョンが大きすぎて足元の実務が疎かになることがある。「今週やるべきこと」を3つだけ書き出す習慣を持つと、ビジョンに"地に足"がつく。

### 宮本武蔵型（E最高）
- **キャッチ**: 「千日の稽古を鍛とし、万日の稽古を練とす。実行だけが真実。」
- **解説**: 考えるより先に手が動く。他の人がまだ準備している間に、もう最初の一手を打っている。
- **注意点**: 実行が速すぎて「これ本当にやるべきだった？」と後から気づくことがある。動く前の"10秒だけ"考える習慣を入れると、実行力はさらに研ぎ澄まされる。

base_salary = 250 + (raw_total / 600) × 1050   // 万円（250〜1300）

// 年齢補正
age_adjusted = base_salary × age_coefficient

// 職種補正（NEW: 職種ごとの年収水準を反映）
job_adjusted = age_adjusted × job_coefficient

// Q26補正（キャリア志向）
q26_adjusted = job_adjusted × q26_coefficient

// SL3によるレンジ生成
salary_center = q26_adjusted

if SL3 ∈ [0, 25]:    // 超安定志向
  salary_min = salary_center - 50
  salary_max = salary_center + 50

if SL3 ∈ [26, 50]:   // やや安定
  salary_min = salary_center - 100
  salary_max = salary_center + 100

if SL3 ∈ [51, 75]:   // やや攻め
  salary_min = salary_center × 0.8
  salary_max = salary_center × 1.3

if SL3 ∈ [76, 100]:  // フルリスクテイカー
  salary_min = salary_center × 0.6
  salary_max = salary_center × 1.8

// 最終クリップ
salary_min = MAX(200, ROUND(salary_min / 10) × 10)    // 最低200万、10万単位に丸め
salary_max = MAX(salary_min + 50, ROUND(salary_max / 10) × 10)  // minより最低50万は上
```

## F-2: 係数テーブル

**年齢係数（age_coefficient）：**
> 年齢が上がれば必ず上がるわけではない。45〜49歳でピーク、55歳以降は役職定年・再雇用で下がる設計。

| 年齢 | 係数 | 考え方 |
|---|---|---|
| 18〜19歳 | 0.35 | 学生・アルバイト。実績ほぼなし |
| 20〜22歳 | 0.45 | 新卒前後。ポテンシャルはあるが市場評価低い |
| 23〜25歳 | 0.58 | 若手。伸び盛りだが年収はまだ |
| 26〜29歳 | 0.73 | 中堅手前。スキルが評価され始める |
| 30〜34歳 | 0.88 | 中堅。スキルが年収に直結 |
| 35〜39歳 | 1.00 | 基準値。マネジメント層が増える |
| 40〜44歳 | 1.10 | ピーク手前。管理職・専門職の年収上昇 |
| 45〜49歳 | 1.15 | ピーク。経験値MAX＋役職の最高地点 |
| 50〜54歳 | 1.10 | 高止まり。ただしスキル低いと頭打ち |
| 55〜59歳 | 0.95 | 役職定年・専門職移行で下がる人が多い |
| 60〜65歳 | 0.78 | 再雇用・嘱託。年収は大幅ダウンが現実 |

**職種係数（job_coefficient）：** NEW
> 職種ごとの市場年収水準を反映。能力値（スコア）が低ければ係数が高くても年収は上がらない。

| 職種 | 係数 | 考え方 |
|---|---|---|
| 経営者・役員 | 1.45 | 最高天井。ただし低スコアなら600万台も出る |
| コンサルタント・士業 | 1.25 | 専門性プレミアム。実力勝負 |
| 金融・保険・不動産 | 1.15 | 業界プレミアム |
| エンジニア・技術職 | 1.08 | 技術プレミアム |
| 医療・福祉 | 1.05 | 専門資格分の上乗せ |
| 企画・マーケティング | 1.03 | やや高め |
| 営業・販売 | 1.00 | 基準値 |
| 人事・採用・教育 | 0.93 | やや低め |
| クリエイティブ | 0.93 | 変動大だが平均はやや低い |
| 事務・管理 | 0.83 | 天井が低い |
| 公務員・団体職員 | 0.78 | 安定だが年収は低い |
| その他・複合 | 1.00 | 中立 |

**Q26補正係数（q26_coefficient）：**

| Q26の回答 | 係数 |
|---|---|
| A（起業・経営） | 1.3 |
| B（大企業マネジメント） | 1.1 |
| C（コンサル・専門職） | 1.0 |
| D（フリーランス・リモート） | 0.9 |
| E（成長環境重視） | 1.0 |

## F-3: 計算例

```
例1：28歳、営業、raw_total=350、Q26=A（起業）、SL3=70（やや攻め）

base_salary = 250 + (350/600) × 1050 = 250 + 613 = 863万
age_adjusted = 863 × 0.88 = 759万
job_adjusted = 759 × 1.00 = 759万（営業＝基準）
q26_adjusted = 759 × 1.3 = 987万

SL3=70 → [51,75]ゾーン
salary_min = 987 × 0.8 = 790 → 790万
salary_max = 987 × 1.3 = 1283 → 1280万

表示：「ポテンシャル年収：790万〜1,280万」
```

```
例2：55歳、事務・管理、raw_total=200（低スコア）、Q26=E、SL3=30

base_salary = 250 + (200/600) × 1050 = 250 + 350 = 600万
age_adjusted = 600 × 0.95 = 570万
job_adjusted = 570 × 0.83 = 473万
q26_adjusted = 473 × 1.0 = 473万

SL3=30 → [26,50]ゾーン
salary_min = 473 - 100 = 373 → 370万
salary_max = 473 + 100 = 573 → 570万

表示：「ポテンシャル年収：370万〜570万」 ← 55歳事務で低スコアなら妥当
```

```
例3：45歳、経営者・役員、raw_total=480（高スコア）、Q26=A、SL3=80

job_adjusted = 1254 × 1.45 = 1818万
q26_adjusted = 1818 × 1.3 = 2363万

```

```
例4：55歳、コンサルタント、raw_total=450（高スコア）、Q26=C、SL3=50
